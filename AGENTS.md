# AGENTS.md

This file defines the development guidelines and rules for AI coding agents working on VSMemo (VSCode Extension Project). Please read and adhere to these guidelines before starting any implementation or modification.

---

## 1. Project Overview and Source Layout

VSMemo is a VSCode extension designed to streamline the creation and editing of Markdown memos and tables.

### Key Directory and File Structure
- **`src/`**: The directory hosting all source code.
  - **`src/extension.ts`**: Entry point of the extension. Responsible for command registration, loading settings, and lifecycle management.
  - **`src/dateNoteTemplate.ts`**: Business logic for date-based memo templates (e.g., variable substitution).
  - **`src/sidebarProvider.ts`**: Webview view provider for the Markdown Tools sidebar.
  - **`src/markdownTableUtils.ts`**: Utilities for creating and editing Markdown tables (adding/removing columns/rows, converting CSV to table, etc.).
  - **`src/markdownEditUtils.ts`**: General editing utilities like wrapping text or inserting today's date.
  - **`src/test/`**: Directory for test code.
    - **`src/test/extension.test.ts`**: Extension integration and unit tests (using Mocha, Sinon, and vscode-test).
- **`dist/`**: The output directory for compiled/bundled JavaScript (`extension.js`) managed by `esbuild.js`. **DO NOT edit this directory directly.**
- **`out/`**: The output directory for compiled test files generated by the `compile-tests` script. **DO NOT edit this directory directly.**
- **`package.json`**: Extension metadata, command registrations (`activationEvents`, `contributes.commands`), configuration settings (`contributes.configuration`), dependencies, and development scripts.
- **`esbuild.js`**: Build configuration script utilizing esbuild.
- **`tsconfig.json`**: TypeScript compiler configuration.

## 2. Build, Test, and Validation Commands

The package manager used in this repository is **pnpm**.

- **Install Dependencies**:
  ```bash
  pnpm install
  ```
- **Build / Compile**:
  ```bash
  pnpm run compile
  ```
  Runs type checking (`check-types`), linter (`lint`), and esbuild bundling (`node esbuild.js`).
- **Real-time Development Watch Mode**:
  ```bash
  pnpm run watch
  ```
  Concurrently runs `watch:esbuild` (bundler watch) and `watch:tsc` (TypeScript type check watch).
- **Code Linting**:
  ```bash
  pnpm run lint
  ```
- **Type Checking Only**:
  ```bash
  pnpm run check-types
  ```
- **Run Tests**:
  ```bash
  pnpm run test
  ```
  *Note: You must run `pnpm run compile-tests` before running the tests. Typically, VSCode's Test CLI or standard terminal triggers this flow.*

---

## 3. Coding Conventions and Architecture Boundaries

- **Strict TypeScript**: Adhere to `tsconfig.json` requirements and define proper types. Avoid using `any` type.
- **VSCode API Utilization**: Use native VSCode APIs (`vscode.window`, `vscode.workspace`, `vscode.TextEditor`, etc.) appropriately and optimize processing to keep editor performance high.
- **Separation of UI and Core Logic**: Core business logic (e.g., template substitution, table formatting) must be decoupled from UI layers (like the sidebar provider `sidebarProvider.ts` or UI command handlers) and written as pure, easily testable functions.
- **Saving File Changes**: When creating new files (like date notes), make sure to properly resolve user configuration values, such as `vsmemo.createDirectory`.

---

## 4. Testing Guidance and Strategy

- Test files are located in `src/test/extension.test.ts`.
- Tests are executed in an integration testing environment using `vscode-test` and Sinon mocks.
- When modifying code, always verify that existing tests pass. When adding new commands or features, append corresponding tests to `extension.test.ts`.

---

## 5. Files and Operations Requiring Caution

- **`package.json`**:
  - When adding a new command, ensure it is fully and properly registered under `activationEvents` (if required), `contributes.commands`, and `contributes.menus` (for the command palette, explorer, or sidebar context). Double-check for missing references.
- **Auto-generated / Build Artifacts**:
  - Files under `dist/` and `out/` are continuously overwritten by `esbuild` and `tsc`. Never modify files in these directories directly.
- **Breaking Changes**:
  - Seek explicit user confirmation before introducing major breaking changes (e.g., renaming settings like `vsmemo.createDirectory`, changing the behavior of template variables, or altering existing command signatures).

---

## 6. Agent Workflow and Escalation Rules

- **Minimize Scope of Changes**: Keep changes surgical and confined strictly to the requested task. Do not engage in unsolicited refactoring.
- **Verify Integrity**: Always execute `pnpm run compile` and `pnpm run lint` after modifying code to ensure no type or lint issues are introduced.
- **Escalation Criteria**:
  - Adding new external npm packages (`package.json` updates).
  - Modifying, renaming, or removing configuration settings (`properties` under `contributes.configuration`).
  - Situations where tests fail to boot or run in the local environment, making complete validation impossible.
